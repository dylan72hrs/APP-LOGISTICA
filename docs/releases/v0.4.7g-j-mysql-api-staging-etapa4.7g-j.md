# Release v0.4.7g-j-mysql-api-staging-etapa4.7g-j - MySQL staging, API real y smoke frontend

## Estado general

ETAPA 4.7G-J implementa la primera integracion real preparada para MySQL staging/cPanel, sin credenciales versionadas y manteniendo `localStorage` como modo default del frontend.

## 4.7G - Conexion DB staging

Se actualizo `Connection.php` para abrir PDO MySQL cuando exista configuracion valida mediante variables de entorno o config local no versionada.

La conexion usa:

- `PDO::ERRMODE_EXCEPTION`
- `PDO::FETCH_ASSOC`
- `PDO::ATTR_EMULATE_PREPARES => false`

Se agrego `backend/php-api/tools/check-db.php` como diagnostico CLI no publico.

## 4.7H - Endpoints GET reales

Se implementaron lecturas reales contra MySQL para:

- `GET /warehouses`
- `GET /workers`
- `GET /inventory`

Si la DB no esta configurada, siguen respondiendo error controlado `DATABASE_NOT_CONFIGURED`.

No devuelven mocks.

## 4.7I - POST /consumptions transaccional

Se agregaron:

- `GET /consumptions`
- `GET /consumptions/:id`
- `POST /consumptions`

`POST /consumptions` valida server-side:

- bodega obligatoria y activa
- trabajador obligatorio y activo
- items obligatorios
- cantidad entera positiva
- producto perteneciente a la bodega
- stock suficiente en servidor

La operacion usa transaccion:

- `beginTransaction`
- `SELECT ... FOR UPDATE`
- insert en `consumption_records`
- insert en `consumption_items`
- descuento de `inventory_items.quantity`
- insert en `audit_log`
- `commit` o `rollback`

## 4.7J - Smoke frontend API

`apiRepository` ahora puede leer API real si se activa:

```text
NEXT_PUBLIC_DATA_REPOSITORY=api
NEXT_PUBLIC_API_BASE_URL=<backend-url>
```

El modo default sigue siendo `localStorage`.

`apiRepository` implementa smoke para:

- bodegas
- trabajadores
- inventario
- consumos
- envio de nuevos consumos hacia `POST /consumptions`

## Limitaciones

- No hay credenciales staging disponibles en el repo.
- No se puede confirmar conexion real sin ambiente cPanel/MySQL.
- Auth real sigue pendiente.
- Los endpoints aun no aplican permisos por usuario.
- CORS queda minimo para smoke y debe endurecerse cuando exista auth real.
- `apiRepository` usa la interfaz snapshot actual y queda solo para smoke, no como integracion final de produccion.
