## Que se planifico

Se definio una estrategia segura para configurar MySQL/MariaDB en el backend PHP sin subir secretos a GitHub.

La arquitectura esperada sigue siendo:

- Frontend Next.js estatico.
- Backend PHP separado en cPanel.
- MySQL/MariaDB administrado por cPanel.
- `localStorage` como default hasta que el backend este probado.
- `apiRepository` como stub/preparacion, no activo.

## Variables/configs futuras

Se documentaron las variables necesarias:

- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- `DB_CHARSET`
- `APP_ENV`

## Estrategia segura de credenciales

Orden recomendado:

1. Variables de entorno del hosting si cPanel las soporta.
2. Archivo de configuracion real fuera del repositorio.
3. Archivo no versionado dentro del hosting solo si el proveedor lo obliga.

`DatabaseConfig.example.php` debe seguir siendo solo plantilla sin secretos.

## Diseno futuro de DatabaseConfig y Connection

`DatabaseConfig` deberia:

- Cargar valores desde variables de entorno o archivo local no versionado.
- Validar campos obligatorios.
- Permitir `APP_ENV` como `local`, `staging` o `production`.
- No exponer secretos en errores.

`Connection` deberia:

- Crear PDO con DSN MySQL.
- Usar `ERRMODE_EXCEPTION`.
- Usar `FETCH_ASSOC`.
- Desactivar emulate prepares si corresponde.
- Fallar con excepcion controlada si falta configuracion.
- No loggear password.
