# Release v0.4.3-etapa4.3 — Ocultar módulos fuera de alcance MVP

## Estado general

ETAPA 4.3 lista para revision.

El MVP queda enfocado en bodega, inventario, trabajadores/invitados, registro de consumo, vale/ficha imprimible, Excel y hosting cPanel.

## Cambios realizados

- Se ocultaron de la navegacion los modulos fuera de alcance actual:
  - /projects
  - /restock
  - /admin/users
- Se mantuvieron visibles los modulos del MVP:
  - /dashboard
  - /inventory
  - /workers
  - /warehouses
  - /consumptions
  - /consumption-sheet
  - /reports

## No modificado

- No se borraron paginas, rutas, mocks ni tipos.
- No se modifico el flujo de consumo.
- No se modificaron reportes.
- No se modifico useData.
- No se implemento Firebase, Firestore, Auth real ni roles reales.
- No se agregaron dependencias.

## Nota tecnica

Los modulos ocultos siguen existiendo y pueden accederse por URL directa. Esta etapa solo reduce la navegacion visible del MVP.

## Riesgos pendientes

- Los modulos ocultos aun pueden abrirse directamente por URL porque no hay permisos reales.
- /admin/users mantiene usuarios y contrasena mock.
- /restock mantiene integracion IA fuera de alcance.
- /projects sigue acoplado a consumos y reportes hasta una etapa posterior.
