# Release v0.4.6a-etapa4.6a - Hardening pre-piloto sin backend

## Estado general

ETAPA 4.6A completa como endurecimiento pre-piloto acotado. La app sigue siendo demo avanzada/MVP sin backend, pero se redujeron riesgos visibles antes de una revision con datos de empresa.

## Correcciones aplicadas

- Se mantiene un unico acceso demo local: usuario `admin`, contrasena `admin`.
- `/admin/users` sigue existiendo, pero queda neutralizado sin CRUD ni contrasenas visibles.
- `/register` sigue existiendo, pero queda deshabilitado para el MVP y no simula creacion de usuarios.
- Se reemplazo la impresion con `document.write` por `window.print()` usando el vale/ficha ya renderizado en DOM.
- Se agregaron estilos globales de impresion A4.
- Se agrego sanitizacion basica para celdas Excel que comienzan con `=`, `+`, `-` o `@`.
- La sanitizacion se aplico en importacion y plantillas Excel de inventario y trabajadores.
- Se agregaron headers basicos en `next.config.ts`:
  - `X-Frame-Options: DENY`
  - `X-Content-Type-Options: nosniff`
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Permissions-Policy: camera=(), microphone=(), geolocation=()`

## Auditoria de restos

- `document.write`: corregido en vale y ficha imprimible.
- `innerHTML`: eliminado de los flujos de impresion revisados.
- `@masterdrilling.com`: sin restos en `src`.
- `123456`: quedan apariciones no relacionadas a contrasena, usadas como RUT/workerId mock.
- `TODO_`: quedan placeholders en `src/firebase/config.ts`, porque Firebase no esta implementado en esta etapa.
- `dangerouslySetInnerHTML`: queda en `src/components/ui/chart.tsx` como parte del componente de estilos dinamicos de charts; no se toca en esta etapa.

## Lo que sigue siendo demo

- No existe backend real.
- No existe autenticacion real.
- La sesion sigue dependiendo de `localStorage`.
- Los roles siguen siendo visuales, no seguridad real.
- Los datos siguen siendo mock/locales.
- No se implemento MySQL.
- No se implementaron API routes.
- No se implemento Firebase Auth ni Firestore.

## Pendiente para etapas posteriores

- Definir backend real para piloto, idealmente MySQL/cPanel segun decision comercial/hosting.
- Reemplazar login demo por autenticacion real.
- Eliminar dependencia de `localStorage` para sesion y datos criticos.
- Revisar CSP cuando exista ambiente real y lista definitiva de assets/origenes.
