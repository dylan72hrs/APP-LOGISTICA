# Release v0.4.7e-implementation-etapa4.7e-implementation - Endpoints read-only iniciales backend PHP

## Estado general

ETAPA 4.7E-IMPLEMENTATION agrega la primera capa de endpoints read-only al backend PHP/cPanel aislado.

Los endpoints existen a nivel de router, controller y repository stub. Todavia no conectan MySQL y no devuelven datos mock como si fueran reales.

## Endpoints agregados

- `GET /warehouses`
- `GET /workers`
- `GET /inventory`

## Comportamiento actual

Mientras la base de datos no este configurada, los endpoints read-only responden con error controlado:

```json
{
  "error": {
    "code": "DATABASE_NOT_CONFIGURED",
    "message": "Database backend not configured.",
    "details": {}
  }
}
```

El codigo HTTP esperado es `501`.

## Endpoint existente mantenido

`GET /health` sigue siendo la prueba tecnica del backend:

```json
{
  "status": "ok",
  "service": "app-logistica-api",
  "mode": "setup"
}
```
