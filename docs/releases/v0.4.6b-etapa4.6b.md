# Release v0.4.6b-etapa4.6b - Reportes operativos MVP

## Estado general

ETAPA 4.6B completa como reajuste del modulo `/reports` al alcance operativo MVP. La pantalla deja de estar centrada en proyectos y dimension financiera, y pasa a enfocarse en consumo de EPP por bodega, trabajador, producto y periodo.

## Cambios aplicados

- `/reports` ahora calcula reportes desde consumos registrados, inventario, trabajadores y bodegas.
- Se elimina el reporte principal por proyecto como eje de la pantalla MVP.
- Se elimina la matriz de valores monetarios por proyecto.
- Se mantiene compatibilidad con consumos antiguos que tengan `projectId`, pero el reporte no depende de ese campo.
- Los consumos sin `projectId` se incluyen normalmente.
- `requesterReference` se muestra solo como referencia opcional cuando existe.
- La exportacion Excel operativa usa sanitizacion de celdas mediante `src/lib/excel-security.ts`.
- No se modifico flujo de stock.
- No se modificaron vale ni ficha imprimible.

## Reportes disponibles

- Reporte por bodega.
- Reporte por trabajador.
- Reporte por EPP/producto.
- Reporte por fecha/periodo.
- Detalle operativo exportable.

## Fuera de alcance

- No se implemento backend.
- No se implemento MySQL.
- No se implementaron API routes.
- No se implemento autenticacion real.
- No se implementaron roles reales.
- No se reactivo `/projects` como foco.
- No se reactivo `/restock` IA.
- No se agregaron dependencias.

## Riesgos pendientes

- Los datos siguen siendo mock/locales.
- La sesion sigue usando `localStorage`.
- Los reportes dependen de datos cargados en el navegador.
- La exportacion Excel es cliente-side y no tiene trazabilidad backend.
- La persistencia real para piloto queda pendiente para etapa posterior con backend/MySQL/cPanel.
